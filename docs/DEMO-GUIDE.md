# Demo Guide

## Melhor narrativa para apresentar a Voithos
1. Abra a aplicacao como plataforma desktop para clinica.
2. Mostre a agenda como centro da operacao diaria.
3. Mostre o fluxo de paciente: cadastro, historico e prontuario.
4. Mostre os modulos administrativos: gestao, financeiro e comunicacao.
5. Feche destacando a arquitetura separada entre desktop, backend e engine de mensageria.

## O que enfatizar para recrutador
- produto real com escopo operacional
- arquitetura separada por responsabilidade
- preocupacao com seguranca e sanitizacao para publicacao
- integracao com servico dedicado de WhatsApp
- base preparada para crescer e modularizar mais

## Ordem recomendada de leitura do codigo
1. `README.md`
2. `main.js`
3. `preload.js`
4. `ipc/`
5. `backend/src/`
6. `services/` e `shared/`
7. `whatsapp-engine/`

## Checklist antes do GitHub publico
- revisar se nao ha dados reais nas imagens
- adicionar 4 a 6 screenshots boas
- criar descricao curta do repositorio no GitHub
- publicar em repositorio separado do sistema operacional
- opcional: gravar um video curto de navegacao
